12 /*
13  * This class is designed to perform an n-sided optimization given the signal and
14  * background histograms (TH1) as well as a fitness function (TF2) to evaluate
15  * them against. The sided-ness of an optimization refers to the number of
16  * internal partitions that are used in determining the best intervals based on
17  * the given fitness function.
18  * e.g.: n = 2 -> 0, 1, 2, m
19  * 0 and m represent the first and last bins in the histograms.
20  * The algorithm will adjust the positions of the bins at 1 and 2 so
21  * as to maximize the fitness function. A possible result could be:
22  * 0, 10, 48, m "odd"
23  * The inner bins have been maximally adjusted and the algorithm picks out
24  * the odd numbered intervals (the first and third or 0-10 and 48-m).
25  * The algorithm will automatically check all lower-order n-sides first. (e.g.
26  * if n = 3 it will check n = 1 and n = 2 first). The algorithm returns a TMatrixD
27  * whose rows are the intervals given in units of the x-axis, not bin number.
28  * In the above example the matrix might be (if bin 0->0, 10->50, 48->240, and
29  * m=100->500):
30  * | 0 50 |
31  * | 240 500 |
32  *
33  * Caution: user's responsibility to delete returned matrix
34  *
35  * Usage:
36  * HAL::CutOptimizer co;
37  * TMatrixD *result = co.Optimize(sig, bkg, 2);
38  * The first and second arguments are the signal and background TH1's for a
39  * 2-sided cut specified by the third argument. A fourth argument can specify if
40  * the intervals should be "even" or "odd" (default is both). A user can also give
41  * a rebin value as the fifth argument as well as an xlow and xhigh value to
42  * restrict the range of the bins (entered in units of the x-axis) as sixth and
43  * seventh arguments.
44  * A user specified fitness function can be supplied as an argument to the
45  * constructor or in the SetFitnessFunction. This function can be retrieved with
46  * the GetFitnessFunction function.
47  *
48  * Requirements:
49  * - The fitness function must accept the signal and background values as the
50  * first and second arguments respectively. Look at default constructor for an
51  * example
52  * - The fitness function must not have the name "co-default"
53  * - The signal and background histograms must have the same number of bins.
54  *
55  * */
